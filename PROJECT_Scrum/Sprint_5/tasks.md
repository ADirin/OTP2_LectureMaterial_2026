# 📌 Important Notes for Students

- The primary focus of this course is **localization, software quality assurance, and documentation**. Students must complete and submit **in-class assignments** as part of their project implementation.
- The course emphasizes **Agile and Scrum practices** together with industry tools such as **CI/CD, Docker, and Kubernetes**.
- The **lecturer will act as the Product Owner**. Teams will participate in a **Sprint Review session via Zoom** at the end of each sprint.
- **Attendance at the Sprint Review is mandatory**, and each team member must clearly present their **individual contribution**.
- This course is delivered through **face-to-face lectures**. No additional support will be provided for students who choose not to participate.

# Sprint 5: UI Localization Planning and implementation (10 Points)
## 🎯 Sprint Goal

- Prepare the application for **full multilingual support** by:
  - Localizing the **User Interface**
  - Enabling **future language scalability**

---

# 📋 Objectives & Deliverables

## 1️⃣ User Interface Localization Preparation (2 Points)

**Tasks**

- Ensure all **UI text is externalized** for localization.
- Identify all **labels, text elements, and messages** requiring translation.
- Implement a **language selection feature** in the UI, including the numbers, date and LTR and RTL items.

*Deliverables:*
•	Functional language selectors (Selected languages and create list of strings that should be localized) are integrated into the application.

---

## 2️⃣ Full GUI Localization Implementation (5 Points)

**Tasks**

- Support **at least two non-Latin languages**  
  *(e.g., Arabic, Japanese, Russian)*.
- Verify UI **updates dynamically** when the language changes.
- Ensure correct layout adjustments for **LTR / RTL languages**.

*Deliverables:*
-	Localized UI with 2 non-Latin language options.
-	Documentation of localization framework or library used.

---

## 3️⃣ Sprint and Product Backlog Update (1 Point)

**Tasks**

- Update the product backlog to include localization-related user stories and tasks.
- Define acceptance criteria for localization features.
- Ensure sprint backlog reflects new or modified localization priorities.

*Deliverables:*
- Updated product backlog with localization stories.
- Defined acceptance criteria and estimation for localization tasks. 

---

## 4️⃣ GitHub Repository and Documentation Update (1 Point)

**Tasks**

- Update `README.md` with instructions for:
  - Selecting languages
  - Identify essential localization resources (e.g., translators, localization tools, CMS if applicable).

*Deliverables:*
- Updated GitHub README with localization setup instructions.
- Update the GitHub with new UML diagrams (doc-->diagrams)

---
