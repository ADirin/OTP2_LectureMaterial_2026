Sprint 5: UI Localization Planning and implementation (10 Points)
Sprint Goal
•	Prepare the application for full multilingual support by localizing the user interface and planning and implementation GUI localization to enable future language scalability.
Sprint Duration
•	2 weeks
Objectives & Deliverables
1. User Interface Localization Preparation (2 points)
•	Ensure the entire user interface (UI) is ready for localization (externalize all static text).
•	Identify and document all text elements, labels, and messages requiring translation.
•	Implement language selection functionality in the UI including the numbers, date, and tables.
Deliverables:
•	Functional language selectors (Selected languages and create list of strings that should be localized) are integrated into the application.
2. Full GUI Localization Implementation (5 points)
•	Localize the GUI to support at least two non-Latin languages (e.g., Arabic, Japanese, Russian, etc.).
•	Verify UI elements dynamically update according to the selected language.
•	Ensure layouts adjust correctly for text direction (LTR/RTL support).
Deliverables:
•	Localized UI with 2 non-Latin language options.
•	Documentation of localization framework or library used.
3. Sprint and Product Backlog Update (1 point)
•	Update the product backlog to include localization-related user stories and tasks.
•	Define acceptance criteria for localization features.
•	Ensure sprint backlog reflects new or modified localization priorities.
Deliverables:
•	Updated product backlog with localization stories.
•	Defined acceptance criteria and estimation for localization tasks. 
4. GitHub Repository and Documentation Update (1 point)
•	Update README.md to include instructions for selecting languages and running localized versions.
•	Identify essential localization resources (e.g., translators, localization tools, CMS if applicable).
Deliverables:
•	Updated GitHub README with localization setup instructions.
•	Update the GitHub with new UML diagrams (doc-->diagrams)

5. Sprint Planning & Review Report (1 point)
•	Prepare the Sprint 6 plan, focusing on database localization (schema changes, data model updates for multilingual content).
•	Conduct sprint review documenting progress, challenges, and improvements.
Deliverables:
Update the project’s GitHub (document)
